# Superstore SQL Analysis (Part 1)

## 📌 Project Overview

This project contains SQL queries used to analyze the Superstore dataset
in PostgreSQL.

The queries perform data validation, filtering, aggregation, and
business analysis to generate meaningful insights from the dataset.

## 📊 Dataset

**Dataset:** Superstore Dataset

The dataset includes:

-   Order ID
-   Order Date
-   Customer Name
-   Segment
-   Region
-   Category
-   Product Name
-   Sales
-   Quantity
-   Discount
-   Profit

------------------------------------------------------------------------

## 📋 SQL Tasks Performed

### Data Validation

-   Count total rows
-   Check NULL values
-   Find duplicate Order IDs

### Data Filtering

-   Records from West region
-   Technology category products
-   Sales greater than 500
-   Orders placed during 2017

### Aggregate Analysis

-   Total sales by region
-   Total profit by category
-   Average sales by segment
-   Total quantity sold
-   Top 10 customers by sales
-   Top 10 products by sales
-   Total sales by category
-   Top 10 products by profit
-   Monthly sales trend
-   Profit by region
-   Average discount by category

------------------------------------------------------------------------

## ▶️ How to Run

1.  Open **pgAdmin 4**.
2.  Connect to your PostgreSQL server.
3.  Create or select your database.
4.  Import the Superstore dataset into a table named **superstore**.
5.  Open **Part-1_SuperMarket Dataset.sql**.
6.  Execute the queries using **F5**.
